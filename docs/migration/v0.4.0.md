# v0.4.0 migration ガイド

v0.4.0 は **複数の破壊的変更** を含みます。詳細は [`CHANGELOG.rst`](../../CHANGELOG.rst) の `0.4.0` 節を参照してください。本ドキュメントでは利用者が確実に踏む可能性が高い 3 件を要約します。

## 1. `mapoi_nav_server` → `mapoi_nav2_bridge` rename ([#204](https://github.com/shimz-robotics/mapoi/issues/204))

Nav2 bridge ノードを `mapoi_nav_server` から `mapoi_nav2_bridge` に rename しました。互換 alias なしの破壊的変更です。

| 種別 | 旧 | 新 |
|---|---|---|
| executable / node 名 | `mapoi_nav_server` | `mapoi_nav2_bridge` |
| `mapoi_bringup.launch.yaml` の launch arg | `with_nav_server` | `with_nav2_bridge` |
| C++ class | `MapoiNavServer` | `MapoiNav2Bridge` |
| header include | `mapoi_server/mapoi_nav_server.hpp` | `mapoi_server/mapoi_nav2_bridge.hpp` |
| node 名から auto-derive される ROS 2 services (`/<node>/get_parameters` 等) | `/mapoi_nav_server/...` | `/mapoi_nav2_bridge/...` |

ROS 2 launch は **未知 arg を fatal にせず default を黙って効かせる**ため、下流 launch から `with_nav_server:=false` を渡している場合は意図せず `mapoi_nav2_bridge` が起動する silent regression になりえます。下記 audit を実行してから更新してください:

```sh
grep -rn 'with_nav_server:=\|mapoi_nav_server' --include='*.launch.*'
```

ROS 2 topic / service / 明示宣言 parameter (`maps_path` / `map_name` / `config_file` 等) / Nav2 native action は不変です。

## 2. AMCL adapter を `mapoi_amcl_localization_bridge` へ分離 ([#209](https://github.com/shimz-robotics/mapoi/issues/209))

旧 `mapoi_nav_server` (現 `mapoi_nav2_bridge`) から AMCL adapter を切り出し、独立した `mapoi_amcl_localization_bridge` executable にしました。`mapoi_nav2_bridge` は Nav2 専用 navigation bridge となり、`/initialpose` への publish はもう行いません。

個別 node 配置で起動している場合は `mapoi_amcl_localization_bridge` を併設してください (`mapoi_bringup.launch.yaml` は新 launch arg `with_amcl_localization_bridge` (default `true`) で対応済)。`mapoi_nav_server` から移設されたパラメータ:

- `initial_pose_topic`
- `initialpose_retry_interval_sec`
- `initialpose_retry_max_attempts`
- `initialpose_post_subscribe_republish_count`

新 contract msg: `mapoi_interfaces/LocalizationBackendStatus`、新 topic: `mapoi/localization/backend_status` (`transient_local` + `MANUAL_BY_TOPIC` liveliness、5 s lease)。WebUI / RViz panel は **Localization 用 indicator** を Navigation indicator と独立で表示し、Initial Pose UI を独立に gate します。詳細は [docs/backend-status.md](../backend-status.md) を参照。

## 3. system tag 定義の hardcode 化 ([#191](https://github.com/shimz-robotics/mapoi/issues/191))

system tag (`waypoint` / `landmark` / `pause`) の定義を `mapoi_server/include/mapoi_server/system_tags.hpp` (`mapoi::kSystemTags`) に hardcode し、`mapoi_server/maps/tag_definitions.yaml` ファイルおよび `mapoi_server/maps/` ディレクトリは廃止しました。

`get_tag_definitions` service contract は不変 (3 system tag + user tag)。標準的な利用 (system tag に手を入れない) には影響なし。`tag_definitions.yaml` で system tag の名前 / 説明を override していたユーザーは `kSystemTags` を編集して再ビルドしてください。

副次的な変更:
- `tag_definitions.yaml` 不在 / malformed 時に空 system tag リストで fallback する degenerate path が削除され、常に 3 system tag が compile-time に保証されます
- パッケージ install layout `share/mapoi_server/` から `maps/` ディレクトリが消えました。外部スクリプトで `share/mapoi_server/maps/` を参照していた場合は更新が必要です
